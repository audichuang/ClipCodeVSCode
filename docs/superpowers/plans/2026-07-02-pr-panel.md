# ClipCodeVSCode PR 面板 Implementation Plan

> **For agentic workers:** implement task-by-task; each task ends with `npm test` green.

**Goal:** 在 Snipcode 加 TreeView PR 面板:選 base ref 看 `base..HEAD` 變更、勾選複製(ClipCode 格式)、檢查 remote 落後。

**Architecture:** host `src/`,自己 spawn git CLI(仿 `extension.ts` 的 `readCommittedBatch`)。複製複用 `buildGraphCopyPayload`(hash=`'HEAD'`)。UI clone `HistoryTreeProvider`/`historyView` 的 TreeView 模式。

**Tech Stack:** TypeScript、VS Code extension API、`node:child_process` spawn、`node --test`。

## Global Constraints
- **有 `node --test` 測試套件**(`test/*.test.ts`,13 檔目前全綠)。驗收 = `npm test` 全綠(它先 `tsc` compile 再 `node --test out/test/*.test.js`)。純函式一律寫單元測試,比對既有測試檔風格(`test/clipboardFormat.test.ts` 用 `node:test` + `node:assert`)。
- **剪貼簿格式是跨工具契約**(ClipCode IntelliJ 端解析同格式)。只透過既有 `buildGraphCopyPayload`/`buildGitPayload` 產生文字,**不得**改 `clipboardFormat.ts` 的 header/label/escape marker。
- `base..HEAD` 用 two-dot(`git diff --name-status <base> HEAD`),與 ClipCode 一致。
- **不碰 vendored `graph/`**:git 原語自己 spawn CLI(gitPath 取 `api.git?.path ?? 'git'`,仿 `extension.ts:92-132`)。
- **從 ClipCode codex 複查納入的 4 個教訓(一開始就做對):**
  1. fetch/diff 順序:`remoteStatus(doFetch)`(含 fetch)必須在 `diffNameStatus` **之前**跑,否則 diff 用 fetch 前舊 ref。
  2. 並發 reload race:用 `reloadGeneration` 計數器,apply UI 前檢查 generation 與當前 base 一致。
  3. fetch 狀態:加 `fetchAttempted`;offline 提示只在 `fetchAttempted && !fetched`。
  4. remote ref 名稱:用完整 `origin/xxx`,非去前綴名。
- 分支 `dev`。每 task commit,不 push。`package.json` 已有使用者未 commit 的 `icon` 改動 + `graph/CLAUDE.md` 改動——**別動那些行**,只加 PR 面板的 contributes。
- 判斷成敗看輸出的 `# fail`/`not ok`/tsc error 文字,勿只信 pipe exit code。

### Task V1: branchDiff.ts(git 存取 + 純函式)

**Files:**
- Create: `src/branchDiff.ts`
- Create test: `test/branchDiff.test.ts`

**Interfaces:**
```typescript
export interface DiffFile { path: string; status: string; oldPath?: string }
export interface RemoteStatus {
  ahead: number; behind: number; upstream?: string;
  fetched: boolean; fetchAttempted: boolean;
}
// 純函式(可測):
export function parseNameStatus(stdout: string): DiffFile[]   // 解析 git diff --name-status
export function parseAheadBehind(revListLine: string): { ahead: number; behind: number } | null
                                     // 輸入 "<behind>\t<ahead>"(rev-list --left-right,left=behind right=ahead)
// git 存取(spawn CLI, 全 try/catch 回安全預設):
export async function diffNameStatus(gitPath: string, repoRoot: string, baseRef: string): Promise<DiffFile[]>
export async function remoteStatus(gitPath: string, repoRoot: string, doFetch: boolean): Promise<RemoteStatus>
export async function candidateBaseRefs(gitPath: string, repoRoot: string): Promise<string[]>
```
git 指令:
- diff:`git -C <root> diff --name-status <base> HEAD`(status 字母 `A|M|D|R###|C###`;`parseNameStatus` 取首字母,rename 行有兩個路徑 → path=新路徑, oldPath=舊路徑)。
- upstream:`git -C <root> rev-parse --abbrev-ref --symbolic-full-name @{u}`(得 `origin/main`;失敗 → 無 upstream)。
- ahead/behind:`git -C <root> rev-list --count --left-right @{u}...HEAD` → `"<behind>\t<ahead>"`。
- fetch(doFetch 時):`git -C <root> fetch --no-tags`(靜默;失敗設 fetched=false);`fetchAttempted = doFetch`。
- candidateBaseRefs:`git -C <root> for-each-ref --format=%(refname:short) refs/remotes`(取 `origin/*`;upstream 置頂;去重)。
spawn 仿 `src/extension.ts:92-119`:`spawn(gitPath, ['-C', repoRoot, ...], { env: { ...process.env, GIT_TERMINAL_PROMPT: '0', LC_ALL: 'C' } })`,收集 stdout,error/非零 exit → 安全預設。

  const out = 'M\tsrc/a.ts\nA\tsrc/b.ts\nD\tsrc/c.ts\nR100\tsrc/old.ts\tsrc/new.ts\n';
  const r = parseNameStatus(out);
  assert.deepEqual(r[0], { path: 'src/a.ts', status: 'M' });

### Task V3: PR TreeView 面板(UI + wiring)

**Files:**
- Create: `src/prTreeProvider.ts`、`src/prPanelView.ts`
- Modify: `src/extension.ts`(呼叫 `registerPrPanel(context)`;`export` `makeGraphCopyDeps`)
- Modify: `package.json`(contributes,保留既有 icon/graph 改動)

**Interfaces / 行為:**
- `registerPrPanel(context: vscode.ExtensionContext): void`(clone `src/historyView.ts:15` 的 git API 取得 + repo 選擇骨架)。
- TreeView id `clipcode.prPanel`,`canSelectMany: true`,`{ manageCheckboxStateManually: true }` 讓原生 checkbox 可用。
- `prTreeProvider.ts`:`TreeDataProvider`,節點含檔案樹(clone `src/historyTree.ts` 的 `buildCommitFileTree` 概念,節點帶 `relPath`/`changeType`/`checkboxState`),預設每檔 `TreeItemCheckboxState.Checked`。
- reload(baseRef, doFetch)(**照 Global Constraints 教訓 1+2**):
  ```
  const generation = ++reloadGeneration;
  // 背景(vscode 無 Task.Backgroundable;用 async + withProgress 或直接 await):
  const status = await remoteStatus(gitPath, root, doFetch);   // 先(含 fetch)
  const diff = await diffNameStatus(gitPath, root, baseRef);   // 後
  // apply 前 guard:
  if (generation !== reloadGeneration || currentBaseRef() !== baseRef) return;
  treeView.message = formatBanner(status);                     // banner
  provider.setFiles(diff);                                     // 全勾
  ```
- `formatBanner(status): string`(純函式,放 `prPanelView.ts` 或 `prCopyService.ts` 皆可,**要有單元測試**):
  - `behind>0` → `⚠ origin 有 {behind} 個新 commit,建議 pull`
  - `behind===0 && upstream` → `✓ 與 {upstream} 同步(ahead {ahead})`
  - `!upstream` → `本分支無對應 origin 分支`
  - append `（未能連線 remote，以下為本地快取狀態）` 只在 `fetchAttempted && !fetched`
- Commands(`contributes.commands` + `menus.view/title` navigation,`when: view == clipcode.prPanel`):
  - `clipcode.pr.selectBase` → `QuickPick`(items = `candidateBaseRefs`)→ `reload(pick, false)`
  - `clipcode.pr.fetch` → `reload(currentBase, true)`
  - `clipcode.pr.refresh` → `reload(currentBase, true)`
  - `clipcode.pr.copyAll` → copy 所有勾選;`clipcode.pr.copySelected`(`view/item/context`)→ copy 右鍵選取
  - copy 實作:`const payload = toGraphCopyPayload(root, diff, checkedPaths); const result = await buildGraphCopyPayload(makeGraphCopyDeps(api, settings), payload); await vscode.env.clipboard.writeText(result.text);` 空勾選 → 警告。
